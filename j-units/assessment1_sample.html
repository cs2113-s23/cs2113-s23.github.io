Assessment 1: Writing Test Cases
Instructions

This assessment is designed to be completed in fifty minutes or less. Your code must work on any valid inputs to receive credit.

Complete the template below such that you will provide code to solve the following problems. When you are satisfied with your solution, you should upload your solution to Marmoset. Marmoset will grade your solution within about a minute; you may modify your solution and resubmit while taking this assessment to get a higher score as necessary.

Write test cases in the template below to find the unknown bug in code that performs the following: a function called checkCode which will check if a string entered by the user is a valid item code or not. A valid item code will have the following characteristics:
  • It will start with an integer between 1 and 999 (without preceding zeros).
  • Next, a period will appear.
  • One or more words composed of letters, separated by spaces, will appear. Each word must be capitalized.
  • The last word is followed by one or more spaces.
  • Finally, the code will end with an optional five digit number.
    Code Templates



    Download the code template for this assessment (right mouse click to save the file). Change the file name to be DriverJava.java. It's technically Java code, but you don't have to worry about that. It's contents are as follows:

        public class DriverJava{
        
        static String[] tests = {
        
            "checkCode 1.A ",
            "true",
            //WRITE YOUR TESTS BELOW IN THE SAME FORMAT (USE COPY-AND-PASTE)
            
            //DO NOT CHANGE ANYTHING BELOW THIS LINE        
            " "};
             public static String getTest(int i){
            return tests[i];
        }
        
        public static String getResult(int i){
            return tests[i+1];
        }
        }
                 
    Just follow the format of each test case according to the example:
    • Start a test case with "
    • On the same line, include the function name, followed by a space, followed by the input
    • Finish the first line by adding a ", followed by a comma, followed by a newline.
    • On the next line, you'll include the expected output: start with a ", followed by either true or false, followed by a ", followed by a comma.
    • Don't change any of the other lines in the file! Make sure you're only adding in lines.

    Ask your TA to eyeball the format if you have questions, although Marmoset will let you resubmit your file if there are any typos.
